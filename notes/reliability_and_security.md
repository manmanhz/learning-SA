# 软件可靠性与安全架构

## 1. 软件可靠性 (Reliability)
*   **量化指标**：
    *   MTBF (平均无故障时间)
    *   MTTR (平均维修时间)
    *   可用性 A = MTBF / (MTBF + MTTR)
*   **设计策略**：
    *   **前向恢复**：出错后继续往前走，修正状态。
    *   **后向恢复**：回滚到出错前的检查点 (Checkpoint)。
    *   **N 版本程序设计**：不同团队用不同语言写同一个功能，投票决定结果。

## 2. 系统安全架构 (Security)
*   **零信任架构 (Zero Trust)**：
    *   核心：默认不信任任何流量。
    *   技术：SDP (软件定义边界)、IAM、微隔离。
*   **安全防御技术**：
    *   **对称加密** (AES)：快，用于加密数据。
    *   **非对称加密** (RSA/国密SM2)：慢，用于密钥交换、数字签名。
    *   **数字签名**：保证**完整性**和**不可抵赖性**。

## 🚀 考试提分技巧 (下午题)
当题目问“如何提高系统的安全性”时，话术：
1. 引入 **双因子认证 (2FA)** 增强身份核验。
2. 采用 **HTTPS/TLS** 协议保证传输机密性。
3. 关键业务数据进行 **脱敏处理** 并存储 **审计日志**。
