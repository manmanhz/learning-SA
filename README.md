# 系统架构设计师备考指南 (2026年5月)

## 1. 考试结构分析
*   **综合知识 (上午题)**：75题单选。覆盖面极广（系统集成、法律、数学、基础理论）。
    *   *你的现状*：工作经验能覆盖 30-40%，剩下的基础理论（如可靠性计算、流水线、形式化方法）需要通过刷题找回记忆。
*   **案例分析 (下午一)**：问答题。通常 5 选 3。
    *   *你的现状*：这是你的强项。重点在于掌握“术语”，例如质量属性（可用性、可维护性、安全性等）的专业描述。
*   **论文写作 (下午二)**：2000-3000 字命题作文。
    *   *你的现状*：最难也最简单。需要提前准备 3 个左右你做过的核心项目，并按架构模板（摘要、正文、总结）进行适配。

## 2. 复习策略：三步走

### 第一阶段：找回记忆 (3月) - 刷题为主
*   **动作**：每天刷 50 道往年综合知识真题。
*   **目标**：识别哪些知识点是完全遗忘的（比如：网络协议明细、软件法律法规、运筹学计算）。
*   **重点**：不要死记硬背，重点看解析。

### 第二阶段：架构标准化 (4月) - 案例+论文准备
*   **动作**：阅读架构风格、评估方法（SAAM/ATAM）、数据库设计相关章节。
*   **论文准备**：
    *   梳理 10 年里的 2 个代表性项目。
    *   按照“微服务、云原生、高可用、安全性”等热门方向各准备一段通用的架构设计描述。

### 第三阶段：模拟与冲刺 (5月前两周)
*   **动作**：限时做 3 套近年的全真模拟题。
*   **论文手写练习**：至少手写一篇论文，保持手感（考试是机考还是笔考请关注当年通知，目前大部分地区已改为机考，但字数压力依然在）。

## 3. 推荐资源
*   书籍：官方《系统架构设计师教程（第二版）》。
*   工具：软考刷题小程序（利用碎片化时间）。

## 4. 目录说明
*   `/notes`: 记录易错题和理论知识点。
*   `/past-exams`: 存放往年真题 PDF。
*   `/thesis`: 存放你的论文素材和草稿。
